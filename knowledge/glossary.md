| **Harness deliberation** | A mediated Foundry session where two selected harnesses exchange final-turn artifacts, challenge assumptions, and produce a traceable implementation-ready spec. | **Mediator** | The human authority in a harness deliberation session: records decisions, revises dispositions, and decides when the session can generate a spec. Mediation / facilitation vocabulary, constrained here to a command pane and event log. | moderator (too social), arbiter |
| **Deferred dissent** | A non-blocking disagreement the mediator carries into `design.md` as a named tradeoff instead of resolving or dropping it. Coined for Foundry to preserve dissent without blocking spec generation. | unresolved concern, open question (when non-blocking) |
| **Session storage tier** | A harness deliberation storage class: Tier 1 is the append-only event ledger, Tier 2 is immutable payload storage, and Tier 3 is rebuildable views. | **Template** | A file foundry installs into a consumer repo. Three classes: *verbatim* (byte-identical everywhere, byte-checked), *seed* (copied once, then repo-owned), *generated* (produced per-repo by the bootstrap skill). | boilerplate, scaffold (as a noun) |
| **Seed** | A template copied once at bootstrap and then owned by the consumer repo (board, glossary, doc stubs, rules). Never byte-checked â€” divergence is the point. Database-seeding vocabulary. | starter, stub (as a class name) |
| **Version marker** | The `foundry-template: <name> v<N>` comment stamped into every verbatim template copy; what `/foundry:update` diffs against. | header, watermark |
| **Gate** | A verification command whose recorded PASS is the evaluator for done-ness. The quick gate runs pre-push and in CI; a heavy gate is optional and repo-specific. | check, validation step |
| **Fixture** | A small repo under `evals/fixtures/` that bootstrap evals run against. xUnit vocabulary. | sample repo, test repo |
| **Seeded defect** | A deliberate fault on a fixture branch (failing test, lint error, debt term) that a generated gate must catch. Grades gates by discrimination, not green-ness. Mutation-testing vocabulary. | mutation, fault injection |
| **COE** | Correction of Error: a dated record of a real failure the setup permitted. Closed only by a mechanical change (gate, lint, rule, or eval fixture). | **Interview** | The bootstrap's question pass that supplies repo content: description, vocabulary seed, API surface, gate commands, isolation needs, unit of work for logging. | wizard, questionnaire |
| **Manifest** | `.foundry/manifest.json` in a consumer repo: the plugin version, the convention version, the harness set, plus each installed verbatim file's template name, version, and content hash â€” how `/foundry:update` tells pristine from customized and whether the repo needs migrating. | **Wide event** | One structured record per unit of work carrying identity, release metadata, execution cost, and decision inputs â€” the canonical log line. The logging convention bootstrap installs. Stripe/Honeycomb vocabulary. | log spam (many fragmented lines per request) |
| **Card / Board** | One row of work on `roadmap/ROADMAP.md` / the board itself â€” the single source of truth for cross-spec status. | ticket, issue (for board rows) |
| **Arm** | A navigation condition under test in an eval; the same task runs under each arm (full-load, native Read/Grep, knowledge.py disclosure). Experimental-design / A-B-testing vocabulary. | variant, condition |
| **Gold span** | The document section(s) an answer requires â€” the navigation eval's ground truth, listed in `answer-key.json`. Information-retrieval (gold / ground-truth) vocabulary. | golden answer, expected section |
| **Context recall** | The fraction of an answer's gold spans an arm loaded into context â€” the completeness axis. RAG-evaluation (RAGAS) vocabulary. | coverage |
| **Context cost** | The context tokens a run loads to reach its answer â€” an eval's efficiency axis. | token cost |
| **Decoy** | A near-duplicate but wrong span (or finding) planted in a fixture; an arm whose answer relies on it scores a decoy hit, testing discrimination. Established in the reviewer fixture. | distractor, trap |
| **Concept** | A single unit of knowledge â€” one markdown file under `knowledge/`, carrying a `type`. Open Knowledge Format vocabulary. | doc, document, page |
| **Type** | A concept's class in frontmatter: one of `reference`, `architecture`, `guide`, `decision`. The OKF required field; foundry keeps it a closed set. | kind, category |
| **Convention version** | A monotonic integer in `.foundry/manifest.json`, bumped only by a *convention break* â€” a release that renames templates, moves directories, or changes frontmatter. | **Signal** | One observation the self-improving loop ingests â€” an eval metric, a code/spec-review finding, a dogfood failure, or a GitHub issue â€” carrying a `source_kind`. Its raw payload lives in the gitignored raw zone; only a generic summary and a hash-ref reach the committed ledger. | **Candidate** | A durable, generic problem the loop aggregates from recurring signals, fingerprinted on the normalized cause (`(affected_surface, failure_mode)`, a script-canonicalized closed tuple) â€” the machine-readable, pre-COE sibling of a COE. Lives committed in the ledger. Bug-triage / candidate-fix vocabulary, constrained here to a committed aggregate problem record. | issue (GitHub-overloaded), cluster |
| **Metric** | A numeric eval sample â€” fixture, run, recall, decoy count, pass/fail â€” in the shape `evals/harness/score_review.py` emits. In the S1 v1 store an eval sample enters as a `candidate_observed` with `source_kind=eval`; the dedicated `metric_observed` event and metric-trend views are deferred (not in the v1 closed event set). Telemetry / time-series-metric vocabulary, constrained here to eval samples. | measurement, score (reserved for ranking) |
| **Candidate ledger** | The committed, generic-by-construction store under `.foundry/state/self-improvement/` holding the loop's candidate and metric history as an append-only event ledger plus a rebuildable digest â€” the loop's durable, cross-clone memory; raw signals stay in the gitignored raw zone. | **Redaction gate** | The single writer of the committed candidate ledger: it rejects any committed-bound record carrying a free-text field or a raw-text marker (path, secret, PII), recording `signal_rejected` and writing nothing. Overfit/genericity is an S2 judgment, not this gate; dedup is the fingerprint-fold under the store write lock, not this gate. The structural defense against raw-signal leakage; ships with its own discriminating eval. The telemetry opt-in gate also records `signal_rejected` (reason `telemetry-disabled`), so `signal_rejected` is the shared rejection event of both gates. Reuses *redact* from the harness-deliberation adapter (which redacts account identifiers from raw output); *gate* per the glossary `Gate` term. | sanitizer, scrubber |
| **Telemetry opt-in gate** | The check, ahead of the redaction gate, that refuses a `telemetry` signal unless the `telemetry.enabled` flag is ON; refusal records `signal_rejected` reason `telemetry-disabled`. `telemetry` is the one external-contribution `source_kind`; the five internal sources are always on.
