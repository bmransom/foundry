# Open Knowledge Format (OKF)

Foundry's knowledge base is a Google **Open Knowledge Format** bundle — the format the
[`knowledge`](../SKILL.md) skill's home-selection and types rest on.

## The shape

- **One concept per file** — a markdown file: YAML frontmatter + a free-form body.
- **Frontmatter** — `type`, `title`, `description` (Foundry requires all three), plus a
  `lifecycle` (`current` / `superseded` / `historical`).
- **Reserved files** — `index.md` (a generated catalog for *progressive disclosure*, one line per
  concept) and `log.md` (the dated, newest-first change history).
- **Links are the graph** — a markdown link from A to B asserts a relationship; the relationship
  lives in the prose, not the link.
- **Read by slice** — navigate `index.md` → `knowledge.py outline <concept>` → `section <concept>
  <heading>`; never full-load a file to answer a narrow question.

## The four types (Foundry's taxonomy)

OKF leaves the `type` set to the producer; Foundry fixes four (`knowledge-config.json`):
`reference` (lookup / contract), `architecture` (how it's built, and why), `guide` (a how-to),
`decision` (a dated record — a COE, an experiment, an investigation).

## Differences from OKF

Foundry is a single-repo engineering KB, not a cross-org exchange format, so it **tightens** the
permissive OKF spec (grounded in `scripts/knowledge.py` + `knowledge/knowledge-config.json`):

| Aspect | OKF spec | Foundry |
|---|---|---|
| `type` set | producer-defined; tolerate unknown types | fixed four — an unknown `type` **fails** `knowledge.py check` |
| required fields | only `type` | `title` + `description` + `type` all required |
| conformance | permissive — MUST NOT reject on missing fields, unknown types, broken links | strict — the lint fails on bad frontmatter; the glossary is a hard contract; orphaned skill refs fail |
| staleness | an optional `timestamp` | a `lifecycle` field (`current`/`superseded`/`historical`) marks a superseded claim |
| `log.md` | format defined; append-only **not** required | append-only discipline (newest-first, never rewrite) |
| reserved files | `index.md` MAY be generated or synthesized on the fly | `index.md` + `log.md` reserved; `index.md` deterministically generated by `knowledge.py index` |
| field set | recommends `resource` / `tags` / `timestamp` | `title` / `description` / `type` / `lifecycle` (no `resource`/`tags`/`timestamp`) |

Source: Google's OKF spec (`GoogleCloudPlatform/knowledge-catalog/okf`).
