# OKF alignment ‚Äî design

**Status:** Spec ‚Äî design drafted (2026-06-14) ‚Äî tracked on the [board](../../ROADMAP.md).

Traces every acceptance criterion in `requirements.md`. The change is one **atomic** migration: the tool, its config, every concept's frontmatter, and the byte-identical twins move together, or `knowledge check` and byte-identity break.

## Architecture overview

The knowledge subsystem has four parts, all migrating together:

1. **The bundle** ‚Äî `knowledge/` holds *concepts* (one markdown file each, `type` frontmatter) plus two reserved files (`index.md`, `log.md`).
2. **The tool** ‚Äî `scripts/knowledge.py` (renamed from `docs.py`) reads `knowledge-config.json`, lints concepts, and generates the listing + sidebar.
3. **The site** ‚Äî vitepress, `srcDir = knowledge/`; `index.md` is both the OKF listing and the home.
4. **The steering rule** ‚Äî `.claude/rules/knowledge-conventions.md` (renamed) plus its enforcement, `knowledge check`, in the gate.

Each lives in **two trees kept in lockstep**: `plugins/foundry/templates/{verbatim,seeds}/` (distributed) and foundry's self-host. The byte-identity gate binds the verbatim copies. The **roadmap and features subsystems are explicitly outside** this ‚Äî they get their leftover concept frontmatter stripped (AC-1.6), not migrated.

## Data flow

**`knowledge check`** (the lint, in the gate): load `knowledge-config.json` ‚Üí glob `concept_globs`, minus `reserved_files` and excludes ‚Üí for each concept, parse YAML frontmatter ‚Üí assert non-empty `type` ‚àà `types` and presence of `title`/`description` ‚Üí exit nonzero naming the first failure, else `knowledge check: OK (N concepts)`.

**`knowledge index`** (generation): glob concepts ‚Üí group by `type` ‚Üí emit an `okf_version`-fronted listing (`# heading`, a section per type, `* [Title](/url) - description`) ‚Üí write `knowledge/index.md`. `knowledge check` re-runs generation to a buffer and **fails if the committed `index.md` is stale** (same pattern as a generated-file freshness gate), so the listing can't drift. *(`okf_version` frontmatter added by `okf-listing-fidelity`; non-current concepts trail their type with a lifecycle tag.)*

**Site render**: vitepress reads `srcDir = knowledge/`; `index.md` renders at `/`; the sidebar comes from `knowledge.py sidebar`; `log.md` renders as an ordinary page.

## Data models

- **Concept frontmatter** ‚Äî `type` (required; one of `reference|architecture|guide|decision`), `title`, `description` (required by foundry; OKF treats them as recommended).
- **`knowledge-config.json`** ‚Äî `types`, `concept_globs`, `required_fields: [title, description, type]`, `reserved_files: [index.md, log.md, README.md]`, `okf_version`, `exclude_substrings`, `exclude_prefixes`, `skill_ref_prefixes`. *(`README.md` + `okf_version` added by `okf-listing-fidelity`.)*
- **`index.md`** (OKF ¬ß6 bundle root) ‚Äî an `okf_version`-only frontmatter, then `# heading`; sections; `* [Title](/url) - description`. *(frontmatter added by `okf-listing-fidelity`.)*
- **`log.md`** (OKF ¬ß7) ‚Äî no frontmatter; `## YYYY-MM-DD` headings, newest first; entries `- **Update** ‚Ä¶` / `- **Creation** ‚Ä¶`.
- **README.md** ‚Äî reserved (excluded from the lint and listing) and srcExcluded from the vitepress build, so it carries no dead sidebar link. *(reserved by `okf-listing-fidelity`; supersedes D3.)*
- **Manifest** ‚Äî verbatim entries record `"template": "knowledge"`.

## CLI (the tool's surface)

`knowledge.py {list,check,outline,section,sidebar,index,sync}` ‚Äî unchanged subcommands except the new `index` (generate the listing) and `check` gaining the index-freshness assertion. Invocation in prose/docs: `python3 scripts/knowledge.py ‚Ä¶`.

## Error handling ‚Äî failure modes

- Missing/empty/out-of-enum `type`, or missing `title`/`description` ‚Üí `check` exits nonzero, names the file.
- Stale committed `index.md` ‚Üí `check` fails with a regenerate hint.
- Verbatim twin drift (e.g., only one `knowledge.py` edited) ‚Üí byte-identity fails.
- **Partial migration** (frontmatter flipped but tool not, or one tree only) ‚Üí `check`/byte-identity fail. Mitigation: land atomically (Migration below); the gate is the acceptance test.
- A roadmap/features file that still carries `kind`/`type` after the strip ‚Üí caught by an AC-1.6 grep assertion in the gate or eval.

## Testing strategy

Independent oracle: the checker is exercised by **hand-built fixture concepts** with known-good/bad frontmatter (the oracle is the fixtures, authored independently of the checker's logic) ‚Äî never the checker certifying itself.

| AC | Verification |
|---|---|
| 1.1 / 1.2 | `test_knowledge.py`: a concept missing/empty `type` fails `check`; a valid one passes |
| 1.3 | `index.md`/`log.md` (no frontmatter) do **not** fail `check` |
| 1.4 | `knowledge.py index` emits a no-frontmatter OKF listing; vitepress build renders `/` (CI) |
| 1.5 | a `type` outside the enum fails `check` |
| 1.6 | gate/eval grep: no `roadmap/**`, `features/**` file carries `kind`/`type`; tool scans only `knowledge/` |
| 2.1 | byte-identity (knowledge.py twins); `test_grade.py` asserts verbatim `scripts/knowledge.py` + template `knowledge` |
| 2.2 | grep check: `docs.py`/"the docs tool"/"doc" no longer names the tool/unit/collection |
| 2.3 | `knowledge-conventions.md` exists; context-budget passes both rules |
| 3.1 / 3.2 | `log.md` present (own + seed), reserved (excluded from lint), OKF shape; rule cites it |
| 4.1 | `bootstrap-eval` expectations: `scripts/knowledge.py`, `knowledge/log.md`, `type`-based concepts present |
| 4.2 | byte-identity gate green across both trees |
| 4.3 | `test_score_review.py` uses `type`; `score_review.py` unchanged and still green (it never read the field) |
| 4.4 | `scripts/check-fast.sh` exits 0 |

## Migration

Breaking, atomic, no back-compat shim (zero consumers; the manifest template-name change `docs`‚Üí`knowledge` breaks nothing). One commit, in this internal order so the gate is green at the end:

1. `git mv` the renames (both trees): `docs.py`‚Üí`knowledge.py`, `test_docs.py`‚Üí`test_knowledge.py`, `docs-config.json`‚Üí`knowledge-config.json`, `docs-conventions.md`‚Üí`knowledge-conventions.md`.
2. Update the tool to read `type` + the new config keys; add `index` + the freshness check.
3. Flip every concept's `kind:`‚Üí`type:` (own + seed); strip roadmap/features frontmatter.
4. Regenerate `index.md`; add `log.md` (own + seed).
5. Update config, rules (combine), glossary (Concept/type terms), manifest name, gate label, all prose + spec refs, eval harness + fixtures + reviewer labels.
6. Run `scripts/check-fast.sh` ‚Üí must exit 0. Then re-run `bootstrap-eval`/`reviewer-eval` as the L2/L3 acceptance (AC-4.1/4.3).

## Open design decisions (confirm at the gate)

- **D1** ‚Äî rename `docs-config.json`‚Üí`knowledge-config.json` and keys `doc_globs`‚Üí`concept_globs`, `kinds`‚Üí`types` (full consistency). *Recommend yes.*
- **D2** ‚Äî `index.md` is **generated** by `knowledge.py` with a freshness gate (vs hand-maintained). *Recommend generated.*
- **D3** ‚Äî `README.md` becomes an ordinary concept (`type: reference`, linted+listed); only `index.md`/`log.md` reserved. *Recommend yes.* **Superseded by `okf-listing-fidelity` (2026-06-29):** `README.md` is `srcExclude`d from the vitepress build, so listing it left a dead sidebar link; it is now reserved (excluded from the lint and listing) alongside `index.md`/`log.md`.
